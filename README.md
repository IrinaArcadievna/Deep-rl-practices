## Practice_1 Cross-Entropy Method & Smoothing Modifications

**1. Цель задания (Objective)**
Решение задачи Taxi-v3 из среды Gymnasium с помощью табличного метода кросс-энтропии (Cross-Entropy Method, CEM). Исследование влияния квантиля отбора, а также модификаций алгоритма: сглаживания Лапласа, сглаживания политики и адаптации метода для стохастических сред.

**2. Формализация среды (Environment Setup)**
В качестве полигона используется классическая среда `Taxi-v3`.

* **Пространство состояний (State Space):** Дискретное, `Discrete(500)`. Описывает все возможные комбинации положения такси (сетка 5х5), 4 возможных локаций пассажира (плюс статус "в такси") и 4 пунктов назначения.

* **Пространство действий (Action Space):** Дискретное, `Discrete(6)`. Включает движения по 4 направлениям, а также действия посадки (pickup) и высадки (dropoff) пассажира.

* **Марковский процесс принятия решений (MDP):** Задача решается в табличном виде, где для каждого из 500 состояний поддерживается распределение вероятностей по 6 действиям.

**3. Используемые алгоритмы (Algorithms)**
Основной алгоритм — табличный метод кросс-энтропии. В рамках практического задания реализованы и протестированы следующие модификации:

* **Базовый CEM:** Отбор элитных траекторий на основе заданного квантиля $q$.

* **Сглаживание Лапласа (Laplace Smoothing):** Добавление константы $\lambda$ к счетчикам частот для предотвращения преждевременного зануления вероятностей неисследованных действий.

* **Сглаживание политики (Policy Smoothing):** Обновление политики путем взвешенного суммирования старой политики и новой (элитной), контролируемое коэффициентом $\alpha$.

* **Адаптация для стохастических сред:** Оценка математического ожидания награды путем сэмплирования $M$ пачек по $K$ траекторий для устранения дисперсии стохастической среды.

**4. Результаты экспериментов (Results)**

* **Базовый алгоритм:** Согласно закону больших чисел, для точной оценки средней награды использовалось 5000 траекторий. Оптимальным значением квантиля оказалось $q=0.5$ (50% траекторий считаются элитными), что обеспечило сходимость за ~45 итераций.

* **Сглаживание Лапласа:** В данной среде модификация не привела к улучшению максимальной награды по сравнению с базовым методом. Оптимальные результаты достигаются при крайне малых значениях параметра (например, 0.001).

* **Сглаживание политики:** Показало стабильные и высокие результаты при значениях параметра сглаживания в диапазоне от 0.6 до 0.9.

* **Стохастическая среда:** Наилучший результат достигается при разбиении данных на большое количество пачек с малым числом траекторий в каждой (например, 750 пачек по 6 траекторий). Это дает наибольшую итоговую награду, однако требует большего числа итераций (около 100) для полной сходимости.

## Practice_2. Deep Cross-Entropy Method

### Задача 1. Deep Cross-Entropy Method (Discrete Action Space)

**1. Цель задания (Objective)**
Решение классической задачи нелинейного управления `Acrobot-v1` с использованием нейросетевой реализации метода кросс-энтропии. Оценка влияния архитектуры нейронной сети и гиперпараметров оптимизации на сходимость алгоритма.

**2. Формализация среды (Environment Setup)**
Среда `Acrobot-v1` представляет собой двухзвенный маятник (аналог гимнаста на перекладине), где управление осуществляется только через сочленение (локтевой сустав).

* **Пространство состояний (State Space):** Непрерывное векторное пространство $S \in \mathbb{R}^6$, описывающее тригонометрические функции углов отклонения звеньев (синусы и косинусы) и их угловые скорости.

* **Пространство действий (Action Space):** Дискретное, `Discrete(3)`. Допустимые управляющие воздействия — приложение положительного, отрицательного или нулевого крутящего момента к сочленению.

**3. Используемый алгоритм (Algorithm)**
В качестве аппроксиматора политики $\pi(a|s)$ используется полносвязная нейронная сеть (Multilayer Perceptron), принимающая на вход вектор состояния и выдающая логиты распределения вероятностей действий.

* **Обновление политики:** Выполняется методом обратного распространения ошибки по элитным траекториям.

* **Функция потерь:** Перекрестная энтропия (`CrossEntropyLoss`).

* **Оптимизатор:** Adam.

**4. Результаты экспериментов (Results)**
В ходе работы был проведен глубокий Grid Search для настройки архитектуры и гиперпараметров алгоритма.

* Варьирование размера скрытого слоя выявило, что архитектуры с 128 скрытыми нейронами обеспечивают оптимальный баланс между выразительной способностью сети и скоростью обучения.

* Эксперименты со скоростью обучения (Learning Rate) показали, что алгоритм чувствителен к шагу оптимизатора: наилучшая сходимость достигнута при $lr = 0.1$. Отбор 50% лучших траекторий ($q=0.5$) позволяет надежно обновлять веса сети без преждевременного схождения к локальным оптимумам.

### Задача 2. Deep Cross-Entropy Method (Continuous Action Space)

**1. Цель задания (Objective)**
Адаптация метода кросс-энтропии для решения задач с непрерывным пространством управляющих воздействий на примере системы `Pendulum-v1`.

**2. Формализация среды (Environment Setup)**
Классическая задача нелинейной динамики — перевернутый маятник.

* **Пространство состояний (State Space):** Непрерывное $S \in \mathbb{R}^3$, представляющее косинус и синус угла отклонения, а также угловую скорость маятника.

* **Пространство действий (Action Space):** Непрерывное `Box(-2.0, 2.0, (1,))` — крутящий момент, применяемый к основанию маятника.

**3. Используемый алгоритм (Algorithm)**

* В данной модификации выход нейронной сети интерпретируется как детерминированное действие (математическое ожидание управляющего воздействия).

* **Исследование среды (Exploration):** Для генерации траекторий к выходу сети добавляется аддитивный гауссовский шум $N(0, \epsilon)$, дисперсия которого затухает с течением времени ($\epsilon_{t+1} = \frac{\epsilon_{0}}{t+1}$), что обеспечивает баланс Exploration-Exploitation.

* **Функция потерь:** Среднеквадратичная ошибка (`MSELoss`), используемая для регрессии предсказаний сети на действия из элитных траекторий.

**4. Результаты экспериментов (Results)**
В ходе обучения было отмечено характерное поведение алгоритма: сеть успешно аппроксимирует действия элитной выборки (функция потерь стабильно стремится к нулю), однако общая награда агента растет крайне медленно. Данный результат подтверждает теоретический тезис о том, что базовая архитектура CEM часто оказывается неэффективной для задач с непрерывным пространством действий. Это обосновывает необходимость применения для подобных систем (UAV, Robotics) более продвинутых архитектур, таких как Actor-Critic (DDPG, SAC, PPO).

## Practice_3. Dynamic Programming. Policy and Value Iterations

**1. Цель задания (Objective)**
Решение задачи поиска оптимальной политики в среде FrozenLake методами динамического программирования. Исследование влияния фактора дисконтирования и сравнение вычислительной сложности различных подходов к оценке ценности состояний.

**2. Формализация среды (Environment Setup)**
В качестве марковского процесса принятия решений используется среда `FrozenLakeEnv`.

**3. Используемые алгоритмы (Algorithms)**

* **Policy Iteration:** Итеративное улучшение политики через два последовательных шага: оценку текущей политики (Policy Evaluation) и ее жадное улучшение (Policy Improvement). Математическая оценка q-значений при независимости награды от следующего состояния имеет вид:
$q(s,a) = R(s, a) + \gamma \sum_{s'} P(s'|s,a) v(s')$.

* **Value Iteration:** Итерационное вычисление оптимальной функции ценности напрямую по уравнению Беллмана оптимальности:
$V_{(i+1)}(s) = \max_a \sum_{s'} P(s' | s,a) \cdot [ R(s,a,s') + \gamma V_{i}(s')] = \max_a Q_i(s,a)$.
Извлечение оптимальной политики происходит по формуле:
$\pi^*(s) = \arg \max_a \sum_{s'} P(s' | s,a) \cdot [ R(s,a,s') + \gamma V^*(s')] = \arg \max_a Q^*(s,a)$.

**4. Результаты экспериментов (Results)**

* **Исследование фактора дисконтирования ($\gamma$):** В алгоритме Policy Iteration был проведен перебор гиперпараметра `gamma` в диапазоне от **0.0** до **1.0**. Эксперименты показали, что наивысшее значение средней награды (0.991) достигается при $\gamma = 1.0$.

* **Оптимизация Policy Evaluation:** Применение «теплого старта» (инициализация `v_values` значениями с предыдущего шага вместо нулевого вектора) успешно модифицирует алгоритм и повышает точность итоговых результатов.

* **Сходимость Value Iteration:** Алгоритм итерации по ценности достиг критерия останова $\Delta < 0.001$ за 64 итерации. Метод требует значительно меньшего количества вычислительных операций, так как исключает внутренний цикл оценки политики и решает уравнение Беллмана сразу относительно оптимальных значений $q(s,a)$.

* **Сравнение производительности:** Для корректного сопоставления Policy Iteration и Value Iteration (ввиду наличия внутреннего цикла у первого) результативность измеряется графиками относительно количества прямых обращений к среде, а не по количеству внешних итераций.

!!!


Если говорить о **смысловом и математическом значении** того, что вы сделали в этом ноутбуке, то суть сводится к следующему:

В отличие от предыдущих методов (где агент исследовал среду вслепую методом проб и ошибок), динамическое программирование (DP) в обучении с подкреплением применяется, когда нам **полностью известна модель среды** — то есть заданы все вероятности переходов $P(s'|s,a)$ и функции наград $R(s,a,s')$.

По сути, вы реализовали классические методы поиска оптимального управления для дискретных стохастических процессов. Эти алгоритмы опираются на уравнение Беллмана оптимальности, которое является дискретным аналогом уравнения Гамильтона — Якоби — Беллмана (HJB), применяемого в теории оптимального управления и дифференциальных играх для непрерывных систем.

В своем коде вы исследовали два фундаментальных подхода:

* **Итерация по политике (Policy Iteration):** Это двухшаговый процесс. Сначала алгоритм фиксирует стратегию и оценивает ее (находит функцию ценности $V(s)$), а затем делает стратегию «жадной» относительно этой оценки, чтобы ее улучшить.
* **Итерация по ценности (Value Iteration):** Этот метод работает напрямую с уравнением Беллмана. Он отбрасывает явную оценку политики и итеративно пересчитывает сами ценности состояний $V(s)$ и $Q(s,a)$ до тех пор, пока они не сойдутся к оптимуму. Оптимальная политика затем извлекается автоматически.

Также в своем анализе вы доказали два важных практических момента:

1. **Эффективность Value Iteration:** Вы наглядно показали, что итерация по ценности сходится быстрее и требует меньше вычислительных ресурсов, так как в ней отсутствует тяжелый внутренний цикл оценки каждой промежуточной стратегии.
2. **Теплый старт (Warm Start):** Выяснилось, что если на этапе оценки передавать значения $V(s)$, полученные на предыдущем шаге (а не инициализировать их нулями заново), алгоритм оптимизируется и дает более точные результаты.

### Task 5: Deep Q-Networks (DQN) & Advanced Modifications

**1. Цель задания (Objective)**
Реализация алгоритма Deep Q-Network (DQN) и исследование архитектурных модификаций, направленных на стабилизацию процесса обучения и преодоление проблемы переоценки функции ценности. Проведение бенчмаркинга методов между собой и с базовым Deep CEM.

В качестве полигона выбрана среда с непрерывным пространством состояний и дискретным пространством действий *(укажите здесь вашу среду, например: Acrobot-v1)*.

**3. Используемые алгоритмы (Algorithms)**
Реализован классический алгоритм DQN с использованием Replay Buffer для декорреляции опыта, а также три фундаментальные модификации:

* **DQN с Hard Target Update:** Использование отдельной целевой сети (Target Network) для вычисления TD-target. Веса целевой сети полностью копируются из основной сети (Policy Network) через фиксированное число шагов. Это стабилизирует мишени при оптимизации функции потерь.
* **DQN с Soft Target Update:** Плавное обновление весов целевой сети на каждом шаге с использованием экспоненциального скользящего среднего (Polyak averaging, параметр $\tau$).
* **Double DQN (DDQN):** Разделение процессов выбора оптимального действия и оценки его ценности. Действие выбирается по основной сети, а его ценность оценивается по целевой. Это математически обоснованный способ устранения систематической ошибки переоценки (overestimation bias), свойственной стандартному оператору $\max$ в уравнении Беллмана.

**4. Результаты экспериментов (Results)**
В ходе сравнительного анализа выявлены следующие закономерности:

* **Вычислительная сложность:** При идентичных гиперпараметрах все реализованные модификации продемонстрировали сопоставимое время выполнения, что подтверждает эффективность использования Target Networks без существенных вычислительных накладных расходов.
* **Стабильность и сходимость:** Наилучшие результаты с точки зрения стабильности обучения и итоговой средней награды показали методы с Target Networks (как Hard, так и Soft Update). На финальных эпизодах средняя награда стабилизировалась на высоком уровне (свыше -100).
* **Дисперсия результатов:** Базовый DQN и, в некоторых случаях, Double DQN продемонстрировали более высокую дисперсию значений наград в процессе обучения. Меньшая стабильность базового DQN ожидаема из-за "движущейся мишени" в TD-ошибке, однако итоговые политики во всех случаях сошлись к допустимым оптимальным значениям.
